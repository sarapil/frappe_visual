# CAPS Quick Reference — AI Coder Cheat Sheet

> Pin this file in your project root as `.caps-rules.md`  
> AI coders: Read this BEFORE writing any code.

## Golden Rules

1. **Every sensitive field → FieldCapabilityMap entry**
2. **Every custom button → ActionCapabilityMap entry**
3. **Every `@frappe.whitelist()` → `require_capability()` guard**
4. **Every DocType controller → `on_load` field filtering**
5. **Client-side = UX only. Server-side = real enforcement.**
6. **Never use `frappe.flags` to bypass CAPS.**
7. **Update `CAPABILITY_REGISTRY.md` with every new capability.**

## Copy-Paste Patterns

### New Capability Registration
```python
# In {app}/setup/caps_setup.py → CAPABILITIES list
("field:Customer:phone", "View Customer Phone", "Field", "Customer", "Access to customer phone number"),
```

### Server Guard
```python
@frappe.whitelist()
def my_method():
    from caps.utils.resolver import require_capability
    require_capability('api:MyDocType:my_method')
```

### Client Conditional Button
```javascript
frappe.ui.form.on('MyDocType', {
    async refresh(frm) {
        if (await frappe.caps.has('action:MyDocType:special_button')) {
            frm.add_custom_button('Special', () => { /* ... */ });
        }
    }
});
```

### DocType Controller on_load
```python
def on_load(self):
    from caps.overrides import filter_response_fields
    filter_response_fields(self)
```

### Field Sensitivity Detection
```
Currency/Float + (cost|price|margin|salary|rate|amount) → hide
Phone/Data + (phone|mobile|email|id_number|passport) → mask  
Text + (notes|internal|private|confidential) → hide
```

## Capability Naming
```
field:{DocType}:{fieldname}       → field:Customer:phone
action:{DocType}:{button_label}   → action:Sales Order:Make Delivery
wf:{DocType}:{transition}         → wf:Purchase Order:Approve
api:{DocType}:{method}            → api:Customer:export_data
report:{Module}:{report_name}     → report:Sales:Commission Report
module:{Module}:{section}         → module:HR:Payroll
```

## Bundle = Job Function
```
"Sales Rep Toolkit"    = field:Lead:phone + action:Lead:convert + ...
"Finance Viewer"       = field:*:amount(read_only) + report:Accounting:*
"Team Lead Toolkit"    = all of member bundle + action:*:approve + ...
```

## Assignment Channels (Resolution Order)
```
1. Direct User Capabilities  (UserCapability)
2. Permission Group Members   (groups user belongs to)
3. Role Capability Maps       (user's Frappe roles → CAPS)
```

## Cache Keys
```
caps:user:{email}     → User's resolved capabilities (TTL: 300s)
caps:bundle:{name}    → Expanded bundle items (TTL: 600s)
```
